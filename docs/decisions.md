# Decision Log

This log tracks all major architectural, statistical, modelling, and operational decisions made throughout the lifecycle of the project.

| Date | Decision | Reasoning | Alternatives Considered | Decided By |
| :--- | :--- | :--- | :--- | :--- |
| 2026-07-12 | Project initialization, directories, and documentation setup (Epic 0). | Establish a clear, structured repository layout and document communication/role guidelines before starting data ingestion to ensure project reproducibility and traceability. | None. Standard best practice for project onboarding. | Jacob |
| 2026-07-12 | Target League selection: English Football League (EFL) Championship. | Leverage domain knowledge (team quality, transfer news, injury context), gain a larger sample size (552 matches/season vs. EPL's 380) for backtesting and paper trading, and exploit less modeled/attention-heavy markets where bookmakers put less pricing effort. | English Premier League (EPL), NBA (Basketball), NFL (American Football). | Jacob |
| 2026-07-12 | Ingest League One (E2) historical data alongside Championship (E1). | Championship has heavy promotion/relegation turnover (3 teams per season). Downloading League One data upfront allows rolling-form and head-to-head features (FEAT-1/FEAT-4) to use cross-league history for newly promoted sides instead of nulling them out. | Null out features for newly promoted teams at start of season. | Jacob |
| 2026-07-12 | Soft-null missing odds and fall back to Market Average. | Pinnacle closing odds are missing for ~50% of matches in season 25/26 due to scraping failures at source. We decided to retain match results (keep row count high) but null the odds, using Market Average as the fallback de-vigging baseline in Epic 4. | Exclude matches with missing Pinnacle odds (would drop ~50% of the 25/26 season). | Jacob |
| 2026-07-12 | Hard-exclude invalid/duplicate rows; process postponed on played date. | Missing results, non-numeric goals, or inconsistent full-time results are completely unusable for model training. Duplicate rows skew results. Postponed matches are processed on their actual played date to avoid lookahead bias. | Null out goals/results for inconsistent matches; process postponed matches on originally scheduled date. | Jacob |
| 2026-07-12 | Version control raw and processed CSV files in Git. | Public datasets occasionally undergo silent corrections. Version-controlling raw and processed CSV files ensures byte-identical pipeline reproducibility against fixed, immutable snapshots. | Ignore raw and processed data in Git and rely on live re-downloads (violates local offline reproducibility). | Jacob |
| 2026-07-12 | Map League One stats 1:1 into Championship rolling features. | Promoted teams inherit their previous League One history for start-of-season lookbacks. We accept this unadjusted cross-league bleed as a known baseline model limitation, acknowledging it may artificially inflate promoted team strength relative to established Championship teams. | Develop a division-based quality discount factor (e.g. multiply League One goals by 0.75), which adds model complexity before baseline validation. | Jacob |
| 2026-07-12 | Impute Premier League relegated team cold-starts with prior-season league-wide averages. | Teams relegated from the Premier League have no E1/E2 history. Using prior-season Championship averages (e.g. 1.35 goals scored/conceded, 1.36 points) provides a stable, unbiased baseline lookback before local Championship matches accumulate, preventing lookahead leakage. | Download and parse full Premier League datasets (E0) to extract prior-season stats, which expands data pipeline scope beyond the EFL. | Jacob |
